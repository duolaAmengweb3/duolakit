USE IT
───────────────────────────────────────────────────────────────

In a project with a PRD (typically PRD.md), run:

  /prd-split             # parse PRD → epic/story/task tree → PRD-tree.md
  /prd-estimate          # add Fibonacci points (1, 2, 3, 5, 8, 13 only)
  /prd-export linear     # produce PRD-tree.linear.csv

Pro features (now unlocked):

  /prd-push linear       # direct GraphQL push to Linear (always dry-runs first)
  /prd-push linear --no-dry-run    # actually create issues, idempotent

Requires LINEAR_API_KEY in your env. Get one at:
  https://linear.app/settings/api

Also unlocked: prd-reviewer sub-agent auto-fires on 8+ point stories,
gives a second opinion (confirm / downgrade / propose clean split).

Try the demo:
  Clone the repo, then:
    cd duolakit/03-prd-splitter/examples
    bash smoke-test.sh    # 20 assertions, all should pass

Then in Claude Code (cd into examples/, then):
  /prd-split sample-prd.md
  /prd-estimate

SAFETY
───────────────────────────────────────────────────────────────

- LINEAR_API_KEY is read from env on every call, never written to disk.
- Default behavior is dry-run; you must add --no-dry-run to actually push.
- > 50 issues in one push requires a second explicit confirmation.
