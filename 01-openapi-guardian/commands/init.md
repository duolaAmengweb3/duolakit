# /openapi-init

You are running the `openapi-init` command from the `openapi-guardian` plugin.

Your task: **bootstrap a project for OpenAPI-driven development**. Set up the file layout, install the right dependencies, and create a minimal working example so the user can immediately start adding endpoints.

## Procedure

1. **Detect or ask for framework.**
   - If user passed `express` / `fastify` / `hono` as argument, use that.
   - Otherwise inspect `package.json` `dependencies` to detect Express / Fastify / Hono.
   - If still unclear, ask: "Which framework? (express / fastify / hono)"
   - **Free tier supports Express only.** If user picks Fastify/Hono, gracefully note Pro is required but proceed with read-only suggestion mode (output the files but don't create them).

2. **Confirm the project layout.**

   Default layout (offer to customize):

   ```
   ./
   ├── openapi.yaml              ← spec, source of truth
   ├── src/
   │   ├── server.ts             ← entry point
   │   ├── routes/
   │   │   └── users.ts          ← example handler
   │   ├── types/
   │   │   └── api.d.ts          ← auto-generated, do not edit
   │   └── sdk/                  ← only if --with-sdk
   │       └── index.ts
   └── package.json (updated)
   ```

3. **Create / update files.**

   - `openapi.yaml` — minimal valid spec with one example endpoint (GET /users/:id). **If file exists, do NOT overwrite** — show diff and ask.
   - `src/types/api.d.ts` — placeholder, will be generated by next `/openapi-sync`.
   - `src/server.ts` — minimal Express + JSON middleware + mount routes.
   - `src/routes/users.ts` — one example handler imports from `../types/api.d.ts`.
   - `package.json` — add scripts:
     ```json
     "scripts": {
       "api:types": "openapi-typescript openapi.yaml -o src/types/api.d.ts",
       "api:check": "echo 'Run /openapi-check in Claude Code'",
       "api:sync":  "echo 'Run /openapi-sync in Claude Code'"
     }
     ```
   - `package.json` `devDependencies` — add `openapi-typescript`.

4. **Run `npm install`** (or detect `pnpm` / `bun` / `yarn` based on lockfile and use that).

5. **Generate types once.**
   - Run `npx openapi-typescript openapi.yaml -o src/types/api.d.ts`.

6. **Print quickstart.**

   ```
   ✓ openapi-guardian initialized for Express
   
   Try it now:
     npm run api:types               # regenerate types
     /openapi-check                  # see what's out of sync
     /openapi-sync                   # auto-fix drift
   
   Edit openapi.yaml to add an endpoint, then run /openapi-sync.
   Read more: https://duolakit.pages.dev/01-openapi-guardian
   ```

## Idempotency rules

- Running `init` twice should be safe.
- Never delete user-written code.
- If `src/routes/users.ts` exists with custom logic, leave it alone — just verify it imports from `../types/api.d.ts`.
- If `openapi.yaml` exists, do not overwrite — ask user before touching it.

## Free vs Pro

- Free: Express only. If user passes `fastify` or `hono`, output:

  ```
  ⚠ openapi-guardian Free supports Express only.
  Pro unlocks Fastify + Hono + NestJS scaffolding.
  
  I'll show you what the files would look like, but won't create them. See:
  https://duolakit.pages.dev#pricing
  
  [print the files inline as code blocks for reference]
  ```

- Pro: full scaffolding for all 4 frameworks.

## Safety

- Always `git status` first. If working directory has uncommitted changes, warn user before proceeding.
- Print every file being created with a 3-line preview before write.
- Run `npm install` only after user confirms (small projects can have huge install times).
